**Step 1:** Write coverage doc — honest accounting:

```markdown
# PostgreSQL syntax coverage

This grammar covers the productions actively used by migraine CLI's
lint (`policy.*` + `layout.*` rules). It is NOT a complete Postgres
parser. Edge-case syntax may parse incorrectly or fail entirely.

## ✅ Stable productions

- DDL: `CREATE/ALTER/DROP TABLE`, `CREATE INDEX`, `CREATE SCHEMA`
- DML: `INSERT/UPDATE/DELETE/MERGE` (full grammar; used by no-dml rule)
- `COPY` (FROM/TO discrimination via sub-rule)
- `CREATE TRIGGER` / `CREATE CONSTRAINT TRIGGER` (no-triggers rule)
- `CREATE FUNCTION` / `CREATE PROCEDURE` (no-stored-routines rule)
- CTEs (`WITH ... AS (...)`)
- `EXPLAIN` wrapping any statement
- Comments (`--` line, `/* */` block)

## ⚠️ Incomplete

- `CREATE EXTENSION` — basic form parses; options block partial
- `CREATE EVENT TRIGGER` — production exists, listener not wired
- Window functions inside CTEs — known parser ambiguity, may fail
- Recursive CTEs — works for simple cases; complex recursion untested

## ❌ Unsupported

- `DO $$ ... $$` anonymous blocks (body opaque to walker)
- `LISTEN` / `NOTIFY` / `UNLISTEN`
- `LOCK TABLE`
- psql meta-commands (`\c`, `\i`, `\d`) — these aren't SQL anyway
- `PREPARE` / `EXECUTE` (dynamic SQL strings are opaque)

## Contributing missing syntax

1. File an issue with sample SQL + expected outcome.
2. PR adds a fixture under `test/corpus/missing/` (passes today via
   "expect parse error"; passes after fix via "expect parse success").
3. If grammar source changes needed, regenerate parser via:
   `make grammar` (runs ANTLR4).
```

**Step 2:** Commit:
```bash
git add docs/coverage.md
git commit -m "docs: explicit PG syntax coverage doc"
```

**Step 1: Write the failing test.**

```go
// pkg/postgresql/corpus_test.go
package postgresql_test

import (
    "io/fs"
    "os"
    "path/filepath"
    "strings"
    "testing"

    "github.com/antlr4-go/antlr/v4"
    parser "github.com/thegorangers/migraine-postgresql-grammar/pkg/postgresql"
)

func TestCorpus_StableProductions_ParseClean(t *testing.T) {
    err := filepath.WalkDir("../../test/corpus/stable", func(p string, d fs.DirEntry, err error) error {
        if err != nil || d.IsDir() || !strings.HasSuffix(p, ".sql") {
            return err
        }
        t.Run(filepath.Base(p), func(t *testing.T) {
            data, err := os.ReadFile(p)
            if err != nil {
                t.Fatal(err)
            }
            errs := tryParse(string(data))
            if len(errs) > 0 {
                t.Errorf("stable fixture %s failed parse:\n%v", p, errs)
            }
        })
        return nil
    })
    if err != nil {
        t.Fatal(err)
    }
}

func TestCorpus_UnsupportedProductions_ProduceErrors(t *testing.T) {
    // Inverse: these MUST fail to parse cleanly. If one starts parsing
    // successfully, either the grammar grew support (move to stable/)
    // or there's a silent acceptance bug (parse error swallowed).
    err := filepath.WalkDir("../../test/corpus/unsupported", func(p string, d fs.DirEntry, err error) error {
        if err != nil || d.IsDir() || !strings.HasSuffix(p, ".sql") {
            return err
        }
        t.Run(filepath.Base(p), func(t *testing.T) {
            data, _ := os.ReadFile(p)
            errs := tryParse(string(data))
            if len(errs) == 0 {
                t.Errorf("unsupported fixture %s parsed cleanly — promote to stable/ if intentional", p)
            }
        })
        return nil
    })
    if err != nil {
        t.Fatal(err)
    }
}

// tryParse runs lexer+parser, collects all syntax errors via a listener.
type errCollector struct {
    *antlr.DefaultErrorListener
    errs []string
}

func (e *errCollector) SyntaxError(_ antlr.Recognizer, _ interface{}, line, col int, msg string, _ antlr.RecognitionException) {
    e.errs = append(e.errs, msg)
}

func tryParse(sql string) []string {
    is := antlr.NewInputStream(sql)
    lexer := parser.NewPostgreSQLLexer(is)
    stream := antlr.NewCommonTokenStream(lexer, antlr.TokenDefaultChannel)
    p := parser.NewPostgreSQLParser(stream)

    coll := &errCollector{}
    p.RemoveErrorListeners()
    p.AddErrorListener(coll)
    lexer.RemoveErrorListeners()
    lexer.AddErrorListener(coll)

    _ = p.Root()
    return coll.errs
}
```

### Task 5: Release workflow with cosign

**Files:**
- Modify or create: `.github/workflows/release.yml`

**Step 1:** This is a Go library — no binary artifacts to sign per se. But:
- Sign the source tarball at GitHub Release.
- Generate SBOM of dependencies via syft.

```yaml
name: Release
on:
  push:
    tags: ["v*"]

permissions:
  contents: write
  id-token: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: sigstore/cosign-installer@v3
      - uses: anchore/sbom-action/download-syft@v0
      - name: Generate SBOM
        run: syft . -o spdx-json=sbom.spdx.json
      - name: Sign SBOM (blob mode)
        run: |
          cosign sign-blob --yes \
            --bundle sbom.spdx.json.bundle \
            sbom.spdx.json
      - uses: softprops/action-gh-release@v2
        with:
          files: |
            sbom.spdx.json
            sbom.spdx.json.bundle
          generate_release_notes: true
```

**Step 2:** Tag `v1.1.0-rc1` to dry-run. Verify `cosign verify-blob` succeeds.

**Step 3:** Commit + tag → release-verify (parallel workflow if mirroring others).
